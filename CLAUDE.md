## 🔴 最高铁律: 先读后写 (违反=严重错误)

每次接到任务,**必须**在写代码前:
1. 搜索并读取项目里的 `project_snapshot.md` (如有)
2. 搜索并读取 `AI_DEBUG_HISTORY.md` (如有)
3. 搜索并读取 `.trae/rules/` 或项目根目录下所有规则文件
4. 在回复中明确说出 "已读取XXX规则",证明已执行检查

**绝对禁止:**
- 不看文档直接写代码
- 重复犯文档已记录过的错误
- 忽略用户明确写过的避坑指南

---

## 🚫 Git推送铁律 (违反=严重错误)

### 当用户说"推送Git"、"提交"、"push"或类似指令时:

**❌ 绝对禁止:**
- 调用MCP GitHub API工具
- 询问用户要提交什么
- 分步骤问确认
- 解释原理或让用户选方案

**✅ 正确做法 (一键执行):**
```bash
git add -A
git commit -m "自动更新"
git push origin $(git branch --show-current)
```

### 原则:
1. **不问** - 直接执行
2. **不解释** - 不讲原理
3. **不中断** - 一次做完
4. **不报错** - 遇到问题自己解决,不输出错误堆栈
5. **用命令行** - 不用GitHub API
