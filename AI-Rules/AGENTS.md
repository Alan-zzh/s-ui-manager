## 🔴 编码前强制检查清单 (最高优先级)

每次接到任务,**必须**在写代码前完成以下步骤:

1. **必查快照** - 搜索并读取 `project_snapshot.md` (如有)
2. **必查病历** - 搜索并读取 `AI_DEBUG_HISTORY.md` (如有)
3. **必查规则** - 搜索并读取 `.trae/rules/` 或项目规则目录下的所有文件
4. **必查影响** - 全局搜索修改涉及的文件范围,改A忘B=隐藏Bug

**绝对禁止:**
- 不看文档直接写代码
- 说"这个简单不用看文档"
- 重复犯文档已记录过的错误
- 忽略用户明确写过的避坑指南

**验证机制:** 在回复中必须明确说出 "已读取XXX规则/文档",证明已执行检查流程。

## 🚫 Git推送铁律

当用户说"推送Git"、"提交"、"push"或任何类似指令时:

**禁止:**
- 调用MCP GitHub API工具
- 询问用户要提交什么
- 分步骤问确认
- 解释原理或让用户选方案
- 输出错误堆栈给用户

**正确做法 (一键执行):**
```bash
git add -A
git commit -m "自动更新"
git push origin $(git branch --show-current)
```

**原则:** 不问、不解释、不中断、不报错、用命令行

## 💡 通用行为准则

1. **行动优先** - 用户说做什么直接做,不问"你确定吗?"
2. **报告简洁** - 成功说"已搞定",失败说"遇到问题"
3. **中文注释** - 所有代码注释用中文
4. **不破坏现有功能** - 修改前检查影响范围
5. **改完必更新文档** - 代码改完同步更新文档
