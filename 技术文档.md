# S-UI 面板修复技术文档

## 问题描述
S-UI 面板在配置 CDN 后出现以下问题：
1. 面板登录失败
2. 订阅链接返回 400 错误
3. 服务启动时出现 SQL Scan 错误
4. 无法进行地面更新
5. 入站规则缺少证书配置
6. Hysteria2 缺少 TLS 配置
7. Reality 配置错误

## 根因分析

### 核心问题：数据库 BLOB 字段类型错误
**问题**：S-UI 数据库中的 `inbounds` 和 `clients` 表的 BLOB 字段被错误地存储为字符串类型，导致 GORM ORM 库在读取时无法将字符串转换为 `json.RawMessage` 类型。

**错误日志**：
```
sql: Scan error on column index 4, name "addrs": unsupported Scan, storing driver.Value type string into type *json.RawMessage
```

### 触发原因
- 数据库操作时未正确处理 BLOB 类型字段
- 直接使用字符串插入 BLOB 字段，未进行类型转换
- 客户端配置不完整，缺少必要的协议配置
- TLS 配置缺少证书路径
- Reality 配置缺少正确的密钥和伪装域名

## 解决方案

### 1. 数据库结构修复
**修复数据库 BLOB 字段类型**，确保所有 BLOB 字段正确存储：

- **inbounds 表**：`addrs`、`out_json`、`options` 字段使用 BLOB 类型
- **clients 表**：`config`、`inbounds` 字段使用 BLOB 类型
- **tls 表**：`server`、`client` 字段使用 BLOB 类型

### 2. 入站规则配置
已配置以下入站规则：
- **VLESS WS**：端口 8080，路径 /vless
- **Trojan WS**：端口 8081，路径 /trojan
- **Hysteria2**：端口 4433，支持端口跳跃

### 3. 客户端配置
创建了完整的客户端配置，包含所有协议的必要参数：
- VLESS WS：UUID
- Trojan WS：password
- Hysteria2：password

### 4. TLS 证书配置
- 生成自签名证书：`/usr/local/s-ui/cert/server.pem`
- 证书密钥：`/usr/local/s-ui/cert/server.key`
- 证书 CN：www.bing.com

### 5. Hysteria2 端口跳跃
- 配置 iptables 规则：20000-50000 UDP 端口转发到 4433
- 命令：`iptables -t nat -A PREROUTING -p udp --dport 20000:50000 -j REDIRECT --to-ports 4433`
